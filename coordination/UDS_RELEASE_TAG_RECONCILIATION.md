## Founder Action Checklist for `uds-v0.3.1` Cut (FA-001)

Run these commands **locally** with the org dev private key. These cannot be done by the agent proxy (no binary upload capability).

```bash
#!/usr/bin/env bash
# FA-001: uds-v0.3.1 founder action script
# Run on a machine with: cosign, gh CLI authenticated, docker authenticated to ghcr.io

set -euo pipefail

ORG="szl-holdings"
VERSION="0.3.1"
KEY="<org-dev-private-key.pem>"  # Replace with actual key path

REPOS=("a11oy" "sentra" "amaru" "rosie" "vessels")

for repo in "${REPOS[@]}"; do
  echo "=== Processing $repo ==="

  # 1. Build/tag container
  git clone https://github.com/$ORG/$repo /tmp/szl-$repo
  cd /tmp/szl-$repo
  docker build -t ghcr.io/$ORG/$repo:uds-v$VERSION .
  docker push ghcr.io/$ORG/$repo:uds-v$VERSION

  # 2. Create tarball
  cd /tmp
  tar --zstd -cf $repo-uds-$VERSION.tar.zst -C szl-$repo .
  sha256sum $repo-uds-$VERSION.tar.zst | awk '{print $1}' > $repo-uds-$VERSION.tar.zst.sha256

  # 3. Sign with cosign
  cosign sign-blob --key $KEY \
    $repo-uds-$VERSION.tar.zst \
    --output-signature $repo-uds-$VERSION.tar.zst.sig

  # 4. Create GitHub release uds-v0.3.1
  gh release create uds-v$VERSION \
    --repo $ORG/$repo \
    --title "$repo-uds v$VERSION — Signed Release" \
    --notes "Signed release with cosign attestation. See release plan in docs/UDS_v0.3.1_RELEASE_PLAN.md"

  # 5. Upload 4-asset signed pattern
  gh release upload uds-v$VERSION --repo $ORG/$repo \
    $repo-uds-$VERSION.tar.zst \
    $repo-uds-$VERSION.tar.zst.sig \
    $repo-uds-$VERSION.tar.zst.sha256 \
    $repo-uds-dev.pub

  echo "=== $repo done ==="
done

# uds-mesh goes last (capstone)
echo "=== Processing uds-mesh (capstone) ==="
# ... (update pointer manifest to v0.3.1, sign, upload)
# See uds-mesh_UDS_v0.3.1_RELEASE_PLAN.md for details
```
